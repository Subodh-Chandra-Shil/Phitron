The var, let and const are three keywords to declare a variable in JavaScript. But they come with various features for robust web application development.

1. var keyword: The var keyword was the go-to way to declaring a variable before ES6 introduced.

Scope
  • Function scope With the 'var' keyword variable that are created within a function can only be accessible in the range of that function.
  • Global scope: If the variable is declared in the global space (which doesn't belongs to any function) can be invoked from anywhere in the script.
  • Drawback: Can be redeclare if the variable declared outside function scope in the first place.

    var a = 10;
    console.log(a);
    // variables declared with 'var' keyword can be re-declare
    var a = 'I Love The World';


2. let keyword:

With ES6 update JS adds let keyword that added many features for developing secure and future-proof application.

Scope
  • Block scope With the 'let' keyword variable we can acheive block scope, where variable declared in a block can't be access apart from the block.
  • Can't redeclare a variable once it created. belongs to any function) can be invoked from anywhere in the script.
  • Drawback: Values aren't immuteable.

    let a = 100;
    console.log(a);
    // Can change definition or update value
    let a += 1;


3. const keyword:

Variables declared with 'const' keyword can't be spoiled or updated anytime sooner in the script neither it can redeclare or reassign. This give immutability which ensures less errors.


Scope: const has a block scope.

    const a = "someone";
    console.log(a);
    // can't redeclare, following code gives error
    const a = 'Loitfa I Love You';
    // can't reassign, following code gives error
    a = 'Laila I Love You';

Null usecases
  • Can be used indicating absence of value in a variable. Suppose we have variable declared but no value at current time. These type of variable should have 'null' as value.
  • Can be used for clearing values of a variable or handling missing data.
  • Can be used as a falsy value.
undefined usecases
  • undefined means a variable that has been declared but not been assigned a in it.
  • It's the default value of a variable when it is declared but not assigned a value explicitly.
  • It can be used for accessing non-existent properties.
